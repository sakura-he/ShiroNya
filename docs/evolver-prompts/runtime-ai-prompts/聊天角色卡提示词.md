# app-api 聊天角色卡 Prompt 字段

适用范围：`app-api` 聊天角色卡、会话元数据、chara_card_v3 导入导出。

本文件记录业务运行时会保存或导入的 AI prompt 字段。它们不是 Codex/Evolver 编码代理的系统提示词。

## 数据来源

- `libs/mongodb/src/schemas/character.schema.ts`
- `libs/mongodb/src/schemas/chat-session.schema.ts`
- `apps/app-api/src/modules/chat/character/schemas/v3-character-card.schema.ts`
- `apps/app-api/src/modules/chat/character/character.service.ts`
- `libs/mongodb/A Narrated Life-1.0.json`

## 角色卡字段

`CharacterData` 和 `chara_card_v3` 中与 prompt 相关的字段：

| 字段 | 类型 | 作用 |
| --- | --- | --- |
| `description` | string | 角色描述或人设。 |
| `personality` | string | 人格/性格设定。 |
| `scenario` | string | 场景/背景设定。 |
| `first_mes` | string | 第一条消息或开场白。 |
| `mes_example` | string | 示例对话。 |
| `creator_notes` / `creatorcomment` | string | 创建者备注。 |
| `system_prompt` | string | 角色卡系统提示词。 |
| `post_history_instructions` | string | 历史消息之后的补充指令。 |
| `extensions.depth_prompt.prompt` | string | 在特定上下文深度插入的提示内容。 |
| `extensions.depth_prompt.depth` | number | 插入深度。 |
| `extensions.depth_prompt.role` | `system` / `user` / `assistant` | 插入时使用的消息角色。 |

## 会话 prompt 元数据

`ChatSession.chat_metadata` 中的字段：

| 字段 | 类型 | 作用 |
| --- | --- | --- |
| `note_prompt` | string | 作者注释提示词。 |
| `note_interval` | number | 注释插入间隔。 |
| `note_position` | number | 注释位置。 |
| `note_depth` | number | 注释深度。 |
| `note_role` | number | 注释角色，当前注释写法为 0=系统、1=用户、2=助手。 |
| `timedWorldInfo` | object | 定时世界信息触发配置。 |

## 当前业务边界

- `chat` 聚合模块当前不直接处理 AI 推理。
- `message` 模块负责保存消息、AI 候选回复、swipes 和切换结果。
- `regenerate()` 当前是保存新的候选回复内容，并不在 Service 内调用模型。
- 角色卡导入导出负责内部 Mongo schema 与 `chara_card_v3` 的映射。

## 对 Evolver/Codex 的使用建议

当任务涉及角色卡、AI prompt 组装或模型调用时：

1. 先确认当前是否已经存在模型调用链路。不要假设 message 服务已经调用 OpenAI 或其他 LLM。
2. 保持 `chara_card_v3` 字段映射稳定。
3. 不要把样例角色卡内容作为项目级编码规则。
4. 修改 prompt 字段含义时，同步 DTO、Mongo schema、导入导出映射和模块文档。
5. 如果新增真正的模型请求，应明确消息角色、上下文拼装顺序、token/长度限制和安全过滤边界。
