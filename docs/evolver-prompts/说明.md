# ShiroAdmin Evolver 提示词目录

生成日期：2026-05-10

本目录是从当前仓库规则、AGENTS、模块文档和代码中的 AI prompt 字段整理出的 Evolver/Codex 提示词资料库。目录内容以当前代码事实为准。

整理目标：

- 给 Evolver 提供当前项目真实可用的编码规则。
- 只记录当前可用的提示词口径。
- 把编码代理提示词和业务运行时 AI prompt 字段分开，避免混用。
- 保留必要来源索引，规则变动时可以回溯更新。

## 目录说明

| 目录 | 用途 |
| --- | --- |
| `active/` | 当前编码时可使用的项目规则提示词。 |
| `active/backend/` | NestJS、Prisma、Better Auth、SpiceDB、日志等后端规则。 |
| `active/frontend/` | `admin-web` 前端与权限 UI 规则。 |
| `active/docs-sync/` | 模块文档同步提示词。 |
| `runtime-ai-prompts/` | 业务运行时会保存或生成的 AI prompt 字段，不作为编码代理规则直接使用。 |
| `genes/` | 适合 Evolver 作为任务策略/记忆素材使用的精简 gene 卡片。 |

## 推荐用法

使用 Codex/Evolver 时，优先把这些文件作为当前项目的提示词上下文。

如果手动给 Codex/Evolver 粘贴上下文，推荐顺序：

1. `active/00-项目级编码代理规则.md`
2. 与任务相关的 `active/backend/*` 或 `active/frontend/*`
3. 如果是权限任务，再追加 `genes/权限变更基因.md`
4. 如果是 SpiceDB Watch、Kafka、投影或 `/system/spicedb-data` 对账任务，再追加 `active/backend/SpiceDB投影同步规则.md`
5. 如果是日志、Loki 或 Promtail 任务，再追加 `active/backend/日志与可观测性规则.md`
6. 如果是文档任务，再追加 `active/docs-sync/模块文档同步规则.md`
7. 如果是聊天角色卡或 AI prompt 字段任务，再追加 `runtime-ai-prompts/*`

## 重要边界

- `active/` 是编码代理规则。
- `runtime-ai-prompts/` 是业务数据结构中的 prompt 字段说明，不等价于 Codex/Evolver 的系统提示词。
- 认证以 Better Auth session 为准；权限以 RBAC 和独立 SpiceDB 显式关系授权链路为准。
- `admin-web` 的核心权限按钮读取接口返回的 `viewerCan...` / `meta.viewerCan...` 字段，不从菜单 permission 推导动作按钮。
- 系统关系表的筛选和分页由后端接口处理，前端通过 `GiTable :request` 请求当前页。

## 当前核心结论

- Better Auth 是当前认证与 session 基础。
- RBAC 是 `admin-api` 和 `app-api` 基础授权事实来源；SpiceDB 服务显式关系授权和关系数据运维。
- SpiceDB Watch 投影链路通过 Redpanda/Kafka 进入本地 PostgreSQL 投影表；投影表是读模型，不是最终授权来源。
- Promtail 当前只提升 `app/env/log_type/level/module` 低基数字段，不提升 `user_id`。
- 修改代码前先给方案；改代码后补测试/验证建议。

## 当前来源说明

当前关系表实现口径：

- 角色、用户、用户组、菜单关系表筛选和分页走后端查询接口。
- 关系初始化接口只返回当前对象和 ID 集。
- 前端 `GiTable` 顶部操作使用 `#custom-extra`。
- 关系抽屉通过 `draftUserIds`、`draftUserGroupIds`、`draftRoleIds` 保留草稿选择态。
- system 管理页头部直接使用 Arco `a-page-header`。
- 可用投影表时优先用投影表查询直接关系；需要完整授权语义时继续使用 SpiceDB lookup/check。
