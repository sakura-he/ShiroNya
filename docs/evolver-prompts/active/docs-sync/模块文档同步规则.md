# 模块文档同步提示词

适用范围：修改 `apps/{app}/src/modules/{module}/` 下代码后的文档同步。

## Prompt

完成模块相关代码改动后，建议根据真实代码变更核对对应模块文档是否需要同步。文档更新以本次真实代码变更和当前代码为准，不需要为了每次小改强行创建固定模板文档。

## 建议步骤

1. 根据实际修改文件路径，判断是否属于后端模块：

```text
apps/{app}/src/modules/{module}/
```

2. 查看该模块是否已有 `docs/` 文档，优先更新现有文档。

3. 如果没有文档，先判断这次变更是否足够重要；只有当模块职责、接口、配置、流程或关系图确实需要沉淀时，再考虑创建文档。

4. 更新内容以本次真实代码变更为准，可按需覆盖：

- 模块职责或边界变化。
- 新增、删除或重命名的公开方法/API。
- 业务流程、权限链路、状态版本、缓存或消息流变化。
- 配置项、环境变量、外部依赖变化。
- 已知限制或后续风险。

## 检查清单

- 新增、删除或重命名了方法吗。
- 修改了主流程吗。
- 修改了环境变量或外部依赖吗。
- 修改了权限、认证、日志、缓存或用户状态版本链路吗。
- 文档是否只描述当前代码事实。
- 是否真的需要 Mermaid 图；只有流程或关系复杂时再补。
- 如果发现代码本身可疑但本次不修复，是否需要记录到根目录 `不合理代码清单.md`。
