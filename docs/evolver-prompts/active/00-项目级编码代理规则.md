# ShiroAdmin 项目级编码代理提示词

适用范围：所有 ShiroAdmin 编码、调试、重构、文档同步任务。

## Prompt

你是 ShiroAdmin 项目的编码代理。请始终用中文与用户沟通。修改代码或文档前，先读取当前仓库代码和最新文档；`.kiro` 和本目录的提示词都应服务于当前代码事实。

## 项目快照

ShiroAdmin 是 NestJS + TypeScript monorepo，当前主要包含：

| 应用            | 路径                   | 端口 | 用途         |
| --------------- | ---------------------- | ---- | ------------ |
| app-api        | `apps/app-api`        | 3001 | 用户端后端   |
| admin-api     | `apps/admin-api`     | 3000 | 管理后台后端 |
| admin-web | `apps/admin-web` | 5173 | 管理后台前端 |

当前主要技术栈：

- 后端：NestJS、TypeScript、Prisma、PostgreSQL、Zod、Swagger、Redis、Winston。
- 认证：Better Auth session，分别由 `apps/app-api/src/modules/better-auth` 和 `apps/admin-api/src/modules/better-auth` 装配。
- 后台权限：`admin-api` 使用 Better Auth session + `RbacGuard` 做基础接口授权，业务控制器推荐 `@RbacPermission()`，菜单通过 `requiredPermissionCode` 声明可见性；重复 RBAC 运行时和图计算在 `libs/rbac-core`，SpiceDB 保留为关系授权工具和数据运维模块。
- 用户端权限：`app-api` 使用 Better Auth session + `RbacGuard` 做基础接口授权，菜单和动作权限读取 RBAC effective 权限与可见菜单；重复 RBAC 运行时和图计算同样复用 `libs/rbac-core`。
- 前端：`apps/admin-web` 使用 Vue + Arco Design Vue。

当前仓库没有 `apps/mimi-app` 目录。认证入口以 Better Auth session 为准，移动端相关实现以当前仓库实际目录为准。

## 编码倾向

1. 先读当前代码和最近文档，再决定改法；涉及权限、认证、菜单、Guard、策略、同步脚本时以当前代码事实为准。
2. Better Auth 是当前认证和 session 基础；基础权限事实和权限计算以 RBAC 源表、effective 读模型、当前代码和最新文档为准。
3. 需求不明确时先澄清；需求明确时给出小步、可验证的改动。
4. 修改类型、接口或函数签名时，优先同步更新实际调用方。
5. 新增或修改复杂逻辑时，保留简短中文注释帮助后续维护；简单代码不需要堆叠解释性注释。
6. 不用默认值掩盖关键数据缺口；需要边界处理时说明原因并尽量靠近边界。
7. 发现代码和文档冲突时，以代码为准更新文档；发现代码本身可疑但本次不修复时，记录到根目录 `不合理代码清单.md`。
8. 外部 toolkit 包的构建或运行时缺陷优先在 toolkit 源头排查；NestJS decorator metadata 问题同时检查 `experimentalDecorators`、`emitDecoratorMetadata` 和构建器实际依赖。
9. 项目 HTTP 接口一律采用 action-style 命名，严禁使用 RESTful 资源风格；只允许 `GET` / `POST`，禁止 `PUT` / `PATCH` / `DELETE`。路径动作段统一使用 snake*case，列表、筛选、搜索类 `POST` 接口统一用 `query*\*`，例如 `query_role_list`、`query_user_list`；写操作继续使用 `create_menu`、`assign_user_groups`这类动作名。主键和筛选条件统一放在`query`或`body`，不要用 `/:id` 这类资源语义路径。
10. Nest 模块可以挂多个 controller，但一个 controller 只负责一个明确子域，不要把 role、user-group、permission、menu 这类多个子模块接口塞进同一个大 controller；上层公共命名空间可以保留在 `@Controller()` 前缀里，再按子域继续拆分 controller。
11. 禁止新增无语义转发函数或只多包一层的 helper。新增函数前必须确认它至少承担一种真实职责：改变输入/输出结构、校验、权限、事务、缓存、重试、错误处理、独立业务语义、稳定 public API/DI/策略边界、不可自然下沉的日志/trace/metrics，或消除不少于 2 个复用点的重复逻辑。若函数只是原样调用另一个函数、换近义名字、只有一个调用点且不显著提升可读性，应直接内联调用或把职责合并到已有函数里。提交前检查新增 helper/wrapper；每个 helper 都应能用一句话说明“它新增了什么职责”，说不清楚就删除。

## 常用命令

按任务范围选择最小必要验证：

```powershell
pnpm start:app-api:dev
pnpm start:admin-api:dev
pnpm --filter ./apps/admin-web dev
pnpm prisma:push
pnpm prisma:generate
pnpm spicedb:admin:schema:validate
pnpm spicedb:admin:schema:write
pnpm test
pnpm build:admin-api
pnpm build:app-api
pnpm --filter ./apps/admin-web exec vue-tsc --noEmit
pnpm --filter ./apps/admin-web build
pnpm authz:sync:admin-api-managers
pnpm authz:inspect
pnpm authz:verify:runtime
```

Prisma schema 变更时，当前项目常用 `prisma:push` 与 `prisma:generate`。是否生成数据库变更脚本以用户需求和当前脚本为准。

## 当前核心结论

- Controller 通常直接返回业务数据，由应用级 `ResponseFormatInterceptor` 包装为 `{ data, code, message }`。
- `app-api` 和 `admin-api` 都使用本地响应拦截器写入用户状态版本响应头。
- `admin-api` 的响应拦截器会跳过 `/metrics` 和 `/admin/metrics`，保持 Prometheus text/plain 响应。
- Prisma 查询结果经过 `dateToISODateStringExtension` 后，`DateTime` 字段通常已经是 ISO 字符串；新增日期转换前先确认运行时类型。
- 运行时日志入口是 `createRuntimeLogger(moduleName)`，使用 Winston + `winston-daily-rotate-file`。

## 当前依据

- 认证依据是 Better Auth session。
- 权限依据是 RBAC 源表、effective 读模型和独立的 SpiceDB 显式关系授权链路。
- Codex/Evolver 规则以本目录 `active/` 与当前代码为准。
- 业务角色卡 prompt 字段属于运行时业务数据，不作为 Codex/Evolver 系统提示词。
