# Gene: Frontend Admin Web

## When

修改 `apps/admin-web`。

## Signals

- Vue 页面、系统管理页面、权限页面、Arco Card、form-create 表单、GiTable 表格、关系抽屉、按钮权限、图谱组件、全局样式。

## Instruction

1. 系统页面需要卡片视觉时优先用 Arco `<a-card>`。
2. system 页面头部直接用 Arco `<a-page-header>`，不要恢复 `NaPageHeader`。
3. 不新增手写 card 外壳类。
4. 新增或设计表单时优先使用 `@form-create/arco-design` 和 `<form-create>`。
5. 常规列表、分页和关系明细表格优先使用项目内 `GiTable` 组件。
6. `GiTable` 顶部操作使用 `#custom-extra` 插槽。
7. 关系管理区域使用 Card 的 `#title` 和 `#extra` 插槽组织表达式和 Card 操作。
8. 系统关系表的筛选和分页走后端分页接口；前端通过 `GiTable :request` 加载当前页。
9. 不恢复前端完整候选集分页工具。
10. 关系初始化 DTO 只保留当前对象和选中 ID 集，不恢复 `directUsers`、`assignableUsers`、`visibleUsers` 等完整候选数组字段。
11. 关系抽屉的草稿选择态通过 `draftUserIds`、`draftUserGroupIds`、`draftRoleIds` 传给后端。
12. 抽屉先打开再请求远程数据。
13. 打开新抽屉前清空上一次数据或设置当前 ID。
14. 核心权限动作按钮读取接口返回的 `viewerCan...` 字段，不从菜单 permission 推导。
15. 页面入口和隐藏路由仍使用导航 permissions。
16. 修改全局样式或多个系统页面时运行类型检查和 build。

## Verify

```powershell
pnpm --filter ./apps/admin-web exec prettier --write <changed files>
pnpm --filter ./apps/admin-web exec vue-tsc --noEmit
pnpm --filter ./apps/admin-web build
```
