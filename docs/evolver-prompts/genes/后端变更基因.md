# Gene: Backend Change

## When

后端 NestJS / Prisma / DTO / Service / Controller 改动。

## Signals

- 文件路径包含 `apps/app-api/src`、`apps/admin-api/src`、`libs/common/src`、`libs/prisma-admin/src`、`libs/prisma-app/src`。
- 涉及 Controller、DTO、Prisma schema、Service、过滤器、拦截器、日志、认证。

## Instruction

1. 先读取当前模块代码和模块文档。
2. 先给方案再改代码。
3. Controller 直接返回业务数据，不手动包 `{ data }`。
4. Swagger 使用 `@ApiOkResByZod({ type: XxxDto })`。
5. DTO 使用纯业务 Schema + `createZodDto` 类。
6. 不对 Prisma DateTime 查询结果调用 `.toISOString()`。
7. 改签名时优先同步更新实际调用方。
8. 禁止新增无语义转发函数。若 helper 只是原样调用另一个函数、换近义名字、只有一个调用点且不显著提升可读性，应直接内联；新增 helper 必须能说明它新增了校验、转换、边界、观测、复用或业务语义。
9. 新增/修改复杂函数时保留简短中文注释。
10. 根据风险补测试，bug 修复优先写重现测试。
11. 修改模块后检查是否需要同步 `docs/{module}.md`。
12. 公开接口使用 `@app/common` 的 `Public`；读取会话仍使用 Better Auth 包的 `Session` 装饰器。
13. 涉及 SpiceDB 投影、Kafka、Loki、Promtail 时，同步核对对应 evolver-prompts 规则。

## Verify

按范围选择：

```powershell
pnpm test
pnpm lint
pnpm build:admin-api
pnpm build:app-api
```
