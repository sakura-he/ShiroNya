# Gene: Docs Sync

## When

模块代码被新增、删除、重构，或认证/权限/日志/缓存/配置链路发生变化。

## Signals

- 文件路径匹配 `apps/{app}/src/modules/{module}/`。
- 代码变动涉及 public API、Service 方法、私有辅助方法、流程图、环境变量、权限边界、日志审计。

## Instruction

1. 定位模块文档 `apps/{app}/src/modules/{module}/docs/{module}.md`。
2. 不存在时，先判断变更是否值得沉淀，再考虑创建基础模块文档。
3. 存在则按本次代码变更更新职责、代码清单、类图、流程图、配置、注意事项。
4. 新增/删除方法时，按复杂度决定是否更新类关系图。
5. 修改业务流程时，按需更新流程图或顺序图。
6. 新增私有辅助方法时，只有确实帮助理解时再加入表格。
7. 权限、认证、缓存、日志变化通常需要更新边界说明。
8. 当前文档只写 Better Auth、RBAC 和独立 SpiceDB 显式关系授权口径。
9. 文档与代码冲突时以当前代码为准，并在提示词来源索引中记录冲突修正。
10. 涉及 `docs/` 根目录规则融合时，只保留当前可用规则和必要来源说明；已经过时的实现文档不要继续作为当前依据。

## Verify

- 文档路径与模块路径一致。
- 文档说明与当前代码一致。
- 新增图表可被 Mermaid 解析。
